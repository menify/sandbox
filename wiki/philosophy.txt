аксиома 1:
            Функция всегда должна возвращать требуемое значение.
            Если внутри функции произошла непредвиденная ошибка, должно возвратиться такое занчение,
            при котором выполнение остального кода было бы бессмысленным.
            
            Это не относиться к функциям которые взаимодействуют непосредственно с пользователем.

аксиома 2:
           Функция всегда должна проверять свои входные параметры.
           Эта проверка должна быть заключена в условные директивы компилятора.
           Чем более тчательнее проводиться проверка, тем выше надежность кода.
           Вызывающая функция всегда должна обеспечивать кооректность входных параметров.
           Если она не может это обеспечить, то она должна сама проверять входные параметры и
           не производить вызов в случае неверных параметров.

аксиома 3:
           Разрешается использовать оператор goto, но только когда необходимо быстро и
           надежно выйти из функции.
           Например: Функция несколько раз выделяет системные ресурсы (формирует некий объект),
           вдруг при выделении очередного ресурса произошла ошибка, то самым эффективным будет
           переход по goto.
           void* new_object( int  num)
           {
                
                
                obj = malloc( sizeof(SOME_OBJECT) );
                
                ...
                
                obj->buf = malloc( num << 10 );
                
                ...
                
                obj->in = open_file( "inputfile.txt", O_READ );
                if (obj->in == INVALID_FILE_HANDLE)
                {
                    goto label_fail_open_input_file;
                }
                
                ...
                
                obj->out = open_file( "outputfile.txt", O_WRITE | O_CREATE | O_SYNC, S_IRWX );
                
                ...
                label_fail_open_output_file:
                    close_file( obj->in );
                
                label_fail_open_input_file:
                    free( obj->buf );
                
                label_fail_buf_alloc:
                    free( obj );
                
                label_fail_obj_alloc:
                    return FAIL;
           }
           
           goto великолепно работает во всех подобных случаях, без использования goto
           код бы получился больше и менее надежный.
